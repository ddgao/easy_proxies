# Easy Proxies Domain Context

本上下文定义代理节点租约、订阅代际切换和稳定代理入口使用的统一业务语言。

## Language

**节点（Node）**：
从配置或订阅获得的一个上游代理端点。不同节点不保证具有不同的公网出口 IP。
_Avoid_: IP、出口 IP

**节点标识（Node Key）**：
跨订阅刷新稳定识别同一节点的身份。节点名称变化不产生新的节点标识。
_Avoid_: 节点名称、显示名称

**租约（Lease）**：
从分配成功时开始计时、在有限时间内通过租约 Gateway 于指定租约冲突域中独占使用一个节点的权利。同一节点可以同时属于不同冲突域的多个有效租约；Legacy 入口流量不在该独占范围内。
_Avoid_: 会话、锁、占用记录

**租约令牌（Lease Token）**：
代表一个租约的不可猜测 Bearer 凭据，持有者可以通过稳定代理入口使用该租约。
_Avoid_: Lease ID、Secret、线程 ID

**租约 API 令牌（Lease API Token）**：
授权自动化调用方申请、查询和释放租约的长期机器凭据。它不代表任何具体租约。
_Avoid_: WebUI Session、租约令牌、代理密码

**租约标签（Lease Label）**：
由调用方提供、仅用于监控定位租约的可选非敏感文本。租约标签不参与身份、鉴权、分配顺序或幂等判断。
_Avoid_: Task ID、租约令牌、幂等键

**租约冲突键（Lease Conflict Key）**：
由调用方提供、用于标识哪些业务任务不能共享同一节点的稳定业务标识。其原文属于潜在敏感信息，不作为日志或监控标识，并与租约标签相互独立。
_Avoid_: 租约标签、任务名称、租约令牌

**租约冲突域（Lease Conflict Domain）**：
具有相同租约冲突键的一组租约；同一冲突域内的有效租约不能同时使用同一 Node Key，不同冲突域可以共享节点。
_Avoid_: 调用方身份、租约标签、全局节点占用

**默认租约冲突域（Default Lease Conflict Domain）**：
所有未提供租约冲突键的租约所属的共同冲突域，用于保留旧调用方之间的节点独占语义。
_Avoid_: 空冲突键、Legacy 入口、全局租约

**空闲节点（Idle Node）**：
当前代际中已经验证可用、且未被指定租约冲突域中的有效租约占用的节点。同一节点可以同时是一个冲突域的占用节点和另一个冲突域的空闲节点。
_Avoid_: 健康节点、可见节点

**空闲节点队列（Idle Node Queue）**：
指定租约冲突域中等待租约分配的空闲节点有序集合。节点按在该冲突域中变为可分配状态的先后顺序进入队尾，租约始终从队首获得节点。
_Avoid_: 全局节点队列、无序节点集合、代理池

**就绪节点（Ready Node）**：
在所属运行代际中已经完成验证、可以参与租约分配的节点。节点可能处于就绪状态，但在指定冲突域中因相同 Node Key 已被租约占用而暂时不是空闲节点。
_Avoid_: 在线节点、健康节点

**退休节点（Retired Node）**：
已经从最新订阅移除、不再接受新租约，但仍可在排空代际中服务已有租约的节点。
_Avoid_: 已删除节点、故障节点、黑名单节点

**阻断节点（Blocked Node）**：
由管理员明确禁止使用、在所有运行代际中拒绝新连接和新租约的 Node Key。阻断会使绑定租约进入故障状态，解除后仍需重新验证。
_Avoid_: 退休节点、永久删除节点、冷却节点

**租约等待者（Lease Waiter）**：
空闲节点耗尽时，正在等待获得节点的租约申请。
_Avoid_: 队列任务、阻塞线程

**排空租约（Draining Lease）**：
已经到期或被主动释放、不再接受新连接，但仍等待已有连接结束的租约。排空期间节点在该租约的冲突域中继续由它独占，其他冲突域仍可使用该节点。
_Avoid_: 已释放租约、过期租约

**故障租约（Broken Lease）**：
绑定节点已经通过独立复检确认不可用、因而拒绝新连接且不会恢复为有效状态的租约。
_Avoid_: 失败请求、冷却租约

**租约 Gateway（Lease Gateway）**：
调用方使用租约令牌访问的稳定 HTTP 代理入口。它在运行代际切换期间保持地址不变。
_Avoid_: Sticky 入口、节点端口、轮换池

**运行代际（Runtime Generation）**：
由一次配置或订阅快照构建的一组节点及其运行状态。一个租约在生命周期内始终归属于同一运行代际。
_Avoid_: 实例、版本、批次

**当前代际（Active Generation）**：
唯一允许为新租约分配节点的运行代际。
_Avoid_: 最新代际、当前实例

**候选代际（Candidate Generation）**：
正在构建或验证，尚不允许分配新租约的运行代际。
_Avoid_: 新实例、预发布实例

**排空代际（Draining Generation）**：
不再接收新租约、但继续服务已有租约和连接的运行代际。
_Avoid_: 旧实例、过期代际

**降级代际（Degraded Generation）**：
仍然是当前代际并继续提供租约，但就绪节点数量已经低于最小就绪容量的运行代际。
_Avoid_: 失败代际、不可用代际

**代际提升（Generation Promotion）**：
候选代际满足准入条件后成为当前代际，同时原当前代际进入排空状态的原子变化。
_Avoid_: 重载、重启、切流

**最小就绪容量（Minimum Ready Capacity）**：
候选代际允许提升前必须达到的就绪节点数量下限，包括不存在当前代际的首次启动。当前约定的下限为 50 个就绪节点；其他代际的租约占用不影响该数量，但会影响提升后的空闲节点数量。
_Avoid_: 最小节点数、最小健康数、最小提升容量

**节点复检（Node Recheck）**：
重新验证当前代际中失效或冷却结束的已有节点，不获取订阅，也不创建新的运行代际。
_Avoid_: 刷新、重载

**节点验证（Node Validation）**：
通过节点访问至少一个配置的公网探测目标，并校验预期 HTTP 状态的过程。主目标和可选备用目标任一成功即通过；首轮全部失败时等待 5 秒复测一次。
_Avoid_: 业务请求、出口 IP 检测、节点复检

**订阅刷新（Subscription Refresh）**：
从已配置的远端订阅重新获取并解析最新节点集合。订阅刷新本身不改变当前代际。
_Avoid_: 节点复检、代际切换、重载

**代际构建（Generation Build）**：
根据一次订阅刷新得到的节点集合创建并验证候选代际的过程。
_Avoid_: 订阅刷新、重载
