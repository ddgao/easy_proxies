# Domain Docs

This is a single-context repository.

Before exploring or changing domain behavior, read:

- `CONTEXT.md` for the canonical domain vocabulary.
- Relevant ADRs under `docs/adr/` for accepted architectural decisions.

Use the glossary's canonical terms in issues, plans, tests, logs, and code. If a proposed change contradicts an ADR, surface the conflict explicitly instead of silently overriding the decision.
