# 使用短期节点独占租约

Status: 节点独占范围和分配队列由 ADR-0003 部分取代；其余决定继续有效。

调用方的一个业务任务需要在多个请求之间保持同一节点，而并发业务任务必须使用不同节点。系统为每个任务分配一个短期独占租约：空闲节点通过并发安全的环形队列分配，已出租节点在释放前不得绕回复用，节点释放后回到队尾；空闲节点耗尽时申请按 FIFO 最多等待十秒。租约默认两分钟，由调用方主动释放并由系统在异常退出时超时回收。

## Consequences

- 租约中途不自动切换节点，业务重试属于调用方职责。
- 单次业务目标请求失败只返回当前请求错误并触发独立节点复检，不直接判定节点或租约失效；复检确认节点不可用后租约进入故障状态且不再恢复。
- 两分钟 TTL 从租约分配成功、节点开始独占时计算，不等待第一次代理连接。
- 第一版租约不可自动或显式续租；超过两分钟的业务任务由调用方重新组织，不延长单个节点的独占时间。
- 租约是纯运行时资源，不持久化；进程重启后所有 Lease Token 失效，调用方负责重新申请并重跑任务，不尝试恢复已经中断的连接。
- 租约到期或主动释放后先停止接收新连接；已有连接排空前节点仍保持独占，最多等待 30 秒后强制关闭残留连接。
- 故障租约立即拒绝新连接并进入排空，不等待原 TTL；连接归零后释放 Node Key，占用节点只有在后续复检恢复后才进入当前代际队尾，原故障租约不复活。
- 管理员手动拉黑按 Node Key 跨代际阻断，并立即使绑定租约进入故障排空；解除阻断后节点必须重新验证，不因订阅刷新直接恢复。
- 节点独占范围由稳定 Node Key 决定，不探测或保证目标网站看到的公网出口 IP 唯一。
- 一个加密安全的随机 Lease Token 同时承担租约标识、Gateway 路由键和数据面 Bearer 凭据；服务端只保存其哈希，日志和监控不得展示明文。
- 租约控制接口使用独立的长期 Lease API Token 鉴权，不复用 WebUI Session；Lease API Token 不受单个租约 TTL 影响。
- 调用方释放租约时同时提交 Lease API Token 和 Lease Token，租约令牌通过请求头传递而不进入 URL；重复释放和释放未知租约均按幂等成功处理。
- 第一版租约创建不提供幂等保证；本机控制请求的响应丢失可能留下一个无法使用的租约，由固定 TTL 自动回收，以此保持服务端只保存 Lease Token 哈希。
- 租约申请可携带最多 64 个字符的可选非敏感标签用于监控定位；标签不要求唯一，也不参与分配、鉴权或幂等判断。
- 第一版不设置每租约并发连接数上限；Gateway 只执行进程级全局资源保护，并在监控中展示各租约连接用量。
- 租约运行时跨代际保留 Node Key 的占用关系，避免同一节点同时出租给新旧代际的两个租约；Legacy 入口流量不参与该独占判断。
- 就绪节点和释放节点按变为可分配状态的先后顺序进入空闲节点队列尾部，租约从队首分配；存在等待者时节点直接交给最早的等待者。
