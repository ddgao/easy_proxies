# 按租约冲突域独占节点

Status: accepted

动态调用方自行提供稳定的租约冲突键，具有相同冲突键的租约构成一个租约冲突域。Node Key 只在同一冲突域内独占，不同冲突域可以无限量共享同一节点；系统不注册调用方身份，也不为节点设置跨冲突域的租约容量。这取代 ADR-0001 和 ADR-0002 中按 Node Key 全局独占的分配范围，同时保留 Lease Token、TTL、非幂等创建、Legacy 入口隔离和运行代际等其他决定。

## Consequences

- 未提供、为空或仅包含空白的租约冲突键进入共同的默认租约冲突域，以保持旧调用方之间的节点独占行为。
- 非空租约冲突键去除首尾空白后按大小写敏感的 UTF-8 原文判断，最长 128 字节；服务端只保存哈希，日志、监控和 WebUI 不展示原文。
- 每个活跃冲突域拥有独立的严格 FIFO 空闲节点队列。初始就绪节点保持既定顺序；新验证、恢复可用或完成排空的节点进入队尾，租约始终从队首获得节点。
- 动态冲突域没有有效租约、排空租约和等待者后删除其队列状态；同一冲突键再次出现时从初始节点顺序重新开始。默认冲突域保留队列状态，以延续旧调用方跨请求的严格 FIFO 顺序。
- 租约释放或到期后的排空只维持该冲突域内的 Node Key 占用，其他冲突域仍可使用同一节点。
- 节点复检、故障、阻断和恢复继续按 Node Key 全局生效；复检确认节点不可用时，绑定该节点的所有冲突域租约都进入故障排空。
- 跨运行代际的 Node Key 占用也以租约冲突域为边界；旧代际租约只阻止同一冲突域在新代际重复取得该节点。
