# 使用稳定租约 Gateway 和运行代际切换

Status: 节点占用范围由 ADR-0003 部分取代；其余决定继续有效。

订阅刷新必须在不中断活跃租约的前提下启用新节点。系统采用始终监听独立 HTTP 代理端口的租约 Gateway，并在其后并行管理候选、当前和排空运行代际；新租约只进入当前代际，已有租约继续留在原代际直到释放或过期。该方案取代关闭整个 sing-box 后重建的租约流量路径，代价是刷新期间需要同时维护多个运行代际，但能够预热新节点、原子提升、排空旧连接并在候选失败时保留当前代际。

## Consequences

- 第一版租约 Gateway 只承诺 HTTP 代理、HTTPS CONNECT 和 TCP 出站。
- 租约 Gateway 使用独立端口，现有轮换、Sticky 和多端口入口保持兼容。
- 普通旧入口不获得租约 Gateway 的无中断或节点独占保证；原有 sequential、random、balance、latency、Sticky 和多端口候选集合不因租约占用而改变。
- 启用租约 Gateway 时必须配置有效的健康探测目标，否则配置校验失败并拒绝启动；租约节点不得采用未经验证的乐观可用状态。
- 节点验证支持一个主目标和一个可选备用目标，任一目标返回预期状态即成功；首轮全部失败时等待 5 秒复测一次，第二轮仍全部失败才判定不可用。
- 运行代际的 dialer、健康状态和连接计数必须彼此隔离，不能继续依赖进程级全局注册表。
- 候选代际达到 50 个就绪节点即可提升，其他代际的租约占用不阻止提升；相同 Node Key 在旧租约释放前不得进入新代际的空闲队列。
- 第 50 个节点就绪后立即提升，剩余节点继续后台探测；未完成验证的节点不得参与租约分配。
- 当前代际的就绪节点跌破 50 时进入降级状态、继续使用剩余节点并触发恢复流程，不自动回滚；最小就绪容量是代际准入门槛，不是运行时停机线。
- 当前代际进入降级状态时先复检已有失效节点；复检后仍不足 50 才执行订阅刷新和候选代际构建。完整恢复连续失败时采用有上限的退避重试。
- 定时、降级和手动触发的订阅刷新统一串行执行并共享结果，同一时刻最多存在一个候选代际；运行期间发生的配置变更只合并为一次后续执行。
- 排空代际在租约和连接全部归零时立即关闭；提升后最迟 2 分 30 秒强制关闭残留资源并记录受影响数量。
- 同一时刻最多存在两个运行代际，只允许当前与候选、或当前与排空的组合；排空期间的新刷新仅保留最新订阅快照，等待排空结束后再构建候选代际。
- 最新订阅移除的节点在旧代际中进入退休状态，不再分配新租约但继续服务已有租约；租约结束后随旧代际退出，后续重新出现时必须重新验证。
